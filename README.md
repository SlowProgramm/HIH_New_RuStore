# RuStore In New Vision

Проект нацелен на быстрое формирование основных составляющих любого интернет-магазина, лёгкость в обращении с компонентами реализации, простую развёртку и возможное масштабирование. Также планируется добавление крутых анимаций с использованием скелетон-лоадинга и SPA.

## Библиотеки и фреймворки

В качестве языка программирования выбран язык Python, предлагающий огромный набор встроенных библиотек, нативную поддержу асинхронного программирования, простую и понятную структуру кода, быстроту реализации.

В качестве сердца проекта использован Django, позволяющий быстро прототипировать, сразу получить доступ к огромному количеству фишек администрирования, а также Jinja-темплейтам и отличному структурированию.

Для хранения данных о приложениях и пользователях используется СУБД Sqlite3, встроенная в Django. Её плюсы в данной реализации очевидны — быстрая скорость в условиях небольшого датасета, который есть в прототипе. Также, поскольку Django поддерживает не только эту СУБД, планируется миграция на Postgre, которая предлагает больший спект возможностей и асинхронное управление.

Для оформления UI/UX дизайна используется стандартный набор инструментов любого фронтендера — HTML и CSS, гибкость которых неоспорима. Пусть и существуют более удобные и масштабные методики работы с фронтом сайта, в данном случае для создания показательного pet-проекта большего и не требуется.

## Кратко о логике работы проекта

Django, развёрнутый на сервере, принимает запросы от пользователя и отправляет ему отрендеренный html-ответ. В процессе рендера подставляются значения из базы данных, собираются необходимые данные, оформляются страницы, содержащие заранее неизвестное количество элементов и т.д.

Обработка запроса происходит смешанно — ~80% обрабатывается через Python, ~17% обрабатывается через синтаксис Jinja-templates и ~3% обрабатывает JavaScript-код (для быстродействия некоторых особо чувствительных элементов).

Python выступает в роли "мостика" между html страницей и данными из базы, локальными файлами, а также данными из браузера пользователя (куки-файлы, юзер-эйджент и т.д.) 

## Кратко о навигации

Попав на главную страницу, пользователь может ознакомиться с преимуществом сервиса, а также зарегистрироваться в нём (или войти в существующий аккаунт).

Навигация по сервису осуществляется в меню сверху и внутри некоторых других страниц. На данный момент есть следующие вкладки:
1. Главная — страница, на которой пользователь изначально был
2. Каталог — страница с приложениями
3. Категории — страница, позволяющая быстро найти приложения из необходимой категории
4. Поиск — здесь можно найти приложение по названию и отсортировать их по релевантности или другой нужной характеристике
5. Личный кабинет (значок юзера) — ссылка на регистрацию/вход или преход в личный кабинет, в котором отображаются оставленные отзывы, купленные игры и полученные достижения за выполнение заданий внутри сервиса.
6. Страница приложения — попасть на неё можно нажав на любое (необходимое) приложение. Здесь содержится подробная информация о разработчике, описание приложения, скриншоты (картинки-иллюстрации) к нему, а также секция отзывов. Здесь же можно посмотреть количество скачиваний, рейтинг и вес приложение. И, самое главное, здесь, в будущем, можно будет установить APK файл приложения.
7. Страница разработчика — попасть на неё можно нажав на имя разработчика на странице приложения. Здесь будет отображено био создателя приложений, а также список этих самых приложений.
